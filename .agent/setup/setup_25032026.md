# Oyster-Cut Setup — 25/03/2026

## Setup Steps

```bash
# 1. Install dependencies
yarn install
cd deps/db-sync && yarn install && cd ../..

# 2. First build (generates static/)
yarn watch
# Wait for all 4 builds to complete, then Ctrl+C

# 3. Install Electron dependencies (one-time)
cd static && yarn install && cd ..

# 4. Load env vars
set -a; source .env.open-source.local; set +a

# 5. Deploy worker
scripts/deploy-worker.sh db-sync

# 6. Electron (macOS)
chflags -R nouchg static/js   # required after yarn watch
ENABLE_DB_SYNC_LOCAL=true ENABLE_FILE_SYNC_PRODUCTION=true yarn cljs:release-electron
ENABLE_DB_SYNC_LOCAL=true ENABLE_FILE_SYNC_PRODUCTION=true yarn release-electron

# 7. iOS
ENABLE_DB_SYNC_LOCAL=true ENABLE_FILE_SYNC_PRODUCTION=true yarn sync-ios-release
cd ios/App && fastlane beta && cd ../..

# 8. Android
ENABLE_DB_SYNC_LOCAL=true ENABLE_FILE_SYNC_PRODUCTION=true yarn release-mobile
npx cap sync android
cd android && ./gradlew assembleDebug && cd ..
```

## Issues Encountered

### 1. Missing `shepherd.js` dependency

**Error:** `File not found with singular glob: node_modules/shepherd.js/dist/js/shepherd.min.js`

**Cause:** `gulpfile.js` references `shepherd.js` but the newer upstream `package.json` dropped it.

**Fix:** Remove the reference from `gulpfile.js` (both desktop and mobile sections):
```diff
-        'node_modules/shepherd.js/dist/js/shepherd.min.js',
```

### 2. `marked` package path changed

**Error:** `File not found with singular glob: node_modules/marked/marked.min.js`

**Cause:** `marked` upgraded from v5 to v17. File moved from `marked.min.js` (root) to `lib/marked.umd.js`.

**Fix:** Update `gulpfile.js` (two occurrences — desktop and mobile):
```diff
- 'node_modules/marked/marked.min.js',
+ 'node_modules/marked/lib/marked.umd.js',
```

### 3. `@sqlite.org/sqlite-wasm` path changed

**Error:** File not found for `node_modules/@sqlite.org/sqlite-wasm/sqlite-wasm/jswasm/sqlite3.wasm`

**Cause:** Newer version moved wasm file to `dist/sqlite3.wasm`.

**Fix:** Update `gulpfile.js` (two occurrences — desktop and mobile):
```diff
- 'node_modules/@sqlite.org/sqlite-wasm/sqlite-wasm/jswasm/sqlite3.wasm',
+ 'node_modules/@sqlite.org/sqlite-wasm/dist/sqlite3.wasm',
```

### 4. `inter-ui` font directory renamed

**Error:** `ENOENT: no such file or directory, scandir 'node_modules/inter-ui/Inter (web)'`

**Cause:** Newer `inter-ui` renamed font directory from `Inter (web)` to `web`.

**Fix:** Update `gulpfile.js` (two occurrences — desktop and mobile):
```diff
- () => gulp.src('node_modules/inter-ui/Inter (web)/*.*').
+ () => gulp.src('node_modules/inter-ui/web/*.*').
```
Output directory stays as `Inter (web)` for compatibility with `inter.css` references.

### 5. Missing `@tabler/icons-webfont` dependency

**Error:** `ENOENT: no such file or directory, scandir 'node_modules/@tabler/icons-webfont/fonts'`

**Cause:** `gulpfile.js` references `@tabler/icons-webfont` but it's not in `package.json`.

**Fix:** Remove the reference from `gulpfile.js` (both desktop and mobile sections):
```diff
-        'node_modules/@tabler/icons-webfont/fonts/**',
         'node_modules/katex/dist/fonts/*.woff2',
```

### 6. `deploy-worker.sh` unbound array variable

**Error:** `DEPLOY_ARGS[@]: unbound variable`

**Cause:** `set -u` (strict mode) treats empty arrays as unbound in bash.

**Fix:** In `scripts/deploy-worker.sh`:
```diff
- npx wrangler deploy --config wrangler.local.toml "${DEPLOY_ARGS[@]}"
+ npx wrangler deploy --config wrangler.local.toml ${DEPLOY_ARGS[@]+"${DEPLOY_ARGS[@]}"}
```

### 7. `static/js` immutable flags block `gulp clean`

**Error:** `ENOTEMPTY: directory not empty, rmdir 'static/js'`

**Cause:** `yarn watch` (shadow-cljs) sets macOS immutable flags (`uchg`) on watched files. `gulp clean` can't delete them.

**Fix:** Remove flags before release builds:
```bash
chflags -R nouchg static/js
```
This is needed every time after `yarn watch` has been running.

### 8. `isbinaryfile` crashes during Electron packaging

**Error:** `RangeError: Invalid array length` at `isbinaryfile/lib/index.js` (`Reader.next`)

**Cause:** `@electron/osx-sign` depends on `isbinaryfile@4.0.10` which has a bug in its protobuf-detection code. It crashes when scanning large binary files (e.g., the 20.6MB ONNX WASM file `ort-wasm-simd-threaded.jsep.wasm`).

**Fix:** Add a `resolutions` field in `static/package.json` to force a newer version:
```json
"resolutions": {
  "isbinaryfile": "^5.0.7"
}
```
Then run `cd static && yarn install && cd ..` to apply.

### 9. Codesigning fails over SSH

**Error:** `Failed to codesign your application with code: 1` — certificate not accessible.

**Cause:** macOS Keychain is locked for SSH sessions. `codesign` can't access the Developer ID certificate.

**Fix:** Unlock the keychain before building:
```bash
security unlock-keychain -p "YOUR_LOGIN_PASSWORD" ~/Library/Keychains/login.keychain-db
```
To make it persistent for SSH sessions (one-time):
```bash
security set-key-partition-list -S apple-tool:,apple:,codesign: -s -k "YOUR_LOGIN_PASSWORD" ~/Library/Keychains/login.keychain-db
```
Alternatively, skip signing entirely:
```bash
APPLE_SIGN_IDENTITY="" APPLE_API_KEY_PATH="" yarn release-electron
```

## Root Cause

Issues 1–5 are caused by `oyster-cut` being cloned from a newer upstream Logseq commit than the private repo. The newer `package.json` has updated dependency versions but `gulpfile.js` wasn't updated to match the new file paths.
