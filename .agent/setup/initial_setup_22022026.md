# Logseq Initial Setup - 22/02/2026

All builds use Cloudflare DO sync via deployed worker at `https://<SYNC_WORKER_DOMAIN>`. The purpose of this setup is to create production builds locally — not for code changes or commits.

## Step-by-step Setup

### 1. Install Prerequisites

```bash
brew install clojure/tools/clojure    # ClojureScript compiler (runs on JVM)
brew install borkdude/brew/babashka   # fast Clojure scripting for dev tasks
npm install -g yarn                   # Yarn v1 Classic (1.22.x)
# Node.js >= 22.20.0 (we installed 22.22.0 via nvm)
# Java JDK (we had 21.0.3)
# Mobile: Xcode, CocoaPods (sudo gem install cocoapods), brew install gnu-sed
```

### 2. Install Dependencies

```bash
yarn install                          # root + packages/ui (via postinstall)
cd deps/db-sync && yarn install       # db-sync server deps
```

### 3. First Build — generates static/

```bash
yarn watch                            # compiles ClojureScript + CSS + webpack
# wait for "Build completed" for :app, :db-worker, :inference-worker, :electron
# Ctrl+C after all builds complete
```

This creates `static/` with `package.json`, compiled JS, CSS, and assets.

### 4. Install Electron Dependencies (one-time)

```bash
cd static && yarn install && cd ..
```

Only needed once — `static/node_modules/` survives `yarn clean`.

### 5. Deploy Sync Server to Cloudflare

```bash
# Login to Cloudflare
cd deps/db-sync/worker && npx wrangler login

# Create resources
npx wrangler d1 create logseq-sync-db
npx wrangler r2 bucket create logseq-sync-assets
# Note: R2 must be enabled in Cloudflare Dashboard first (free tier)
# Note: Workers & Pages must be visited once in Dashboard to create workers.dev subdomain
```

Update `deps/db-sync/worker/wrangler.toml` default section with your resource IDs:
```toml
[[d1_databases]]
binding = "DB"
database_name = "logseq-sync-db"
database_id = "<your-d1-id>"

[[r2_buckets]]
binding = "LOGSEQ_SYNC_ASSETS"
bucket_name = "logseq-sync-assets"
```

Run remote migration, build, and deploy:
```bash
npx wrangler d1 migrations apply logseq-sync-db --remote
cd .. && yarn release && cd worker
npx wrangler deploy
```

Our deployment: `https://<SYNC_WORKER_DOMAIN>`

## Running — Development

```bash
# Terminal 1 — web + electron compilation (syncs via deployed Cloudflare worker)
ENABLE_DB_SYNC_LOCAL=true yarn watch

# Terminal 2 — electron
yarn dev-electron-app
```

No local wrangler needed — sync server is always running on Cloudflare.

Modifies: `static/js/`, `static/css/`, `static/electron.js`

## Running — Production

### Build

```bash
# 1. Build web (modifies static/ — runs gulp clean first, then compiles)
ENABLE_DB_SYNC_LOCAL=true yarn release-app

# 2. Build electron — two steps required
#    cljs:release-electron compiles ClojureScript → static/js/, static/electron.js
#    release-electron runs gulp:build (cleans static/), webpack, then electron-forge make
ENABLE_DB_SYNC_LOCAL=true yarn cljs:release-electron
ENABLE_DB_SYNC_LOCAL=true yarn release-electron
cd static && yarn electron:make-macos-arm64 && cd ..
# Output: static/out/make/Logseq.dmg

# 3. Build mobile
ENABLE_DB_SYNC_LOCAL=true yarn sync-ios-release
```

Note: `yarn release-electron` does NOT compile ClojureScript — always run `yarn cljs:release-electron` before it.

### Run

```bash
# Web
npx serve static -l 3001

# Electron — install Logseq.dmg from static/out/make/

# Mobile — npx cap open ios → Cmd+R in Xcode
```

No sync server to manage — it's deployed on Cloudflare.

## Redeploying Sync Server

```bash
cd deps/db-sync && yarn release && cd worker && npx wrangler deploy
```

## Fresh Build (nuclear option)

```bash
yarn clean                            # deletes static/ contents (keeps node_modules)
# Then re-run the build steps above
```
